

## Python
#### Project Python dikatakan Sederhana Jika:  
- Fungsinya fokus dan terbatas (single-purpose).  
- Tidak terlalu banyak dependensi atau konfigurasi.  
- Kode masih bisa dibaca dan dimengerti oleh pemula menengah. 

#### Contoh GUI sederhana:  
- PDF Merger (dengan Tkinter + PyPDF2)  
- Kalkulator GUI  
- To-Do List  
- Stopwatch / Timer  
- File renamer  
- Screenshot tool  

#### Tapi, jika GUI-nya:  
Kompleks, misalnya multi-jendela, drag-and-drop, banyak state logic.  
Menggunakan framework besar (misal, PyQt dengan desain profesional, atau Electron wrapper). Terintegrasi dengan database besar atau API besar, maka sudah masuk kategori menengah ke atas.  

## Web
#### Ciri-ciri Web Project Sederhana:  
- Fungsi Terbatas & Fokus  
- Hanya memiliki 1–3 fitur utama.  
- Contoh: Kalkulator online, form contact, todo list, galeri gambar, anime list statis.  
- Tanpa atau Minim Backend  
- Hanya pakai HTML/CSS/JS, atau backend sangat ringan (misal PHP native atau Express.js tanpa database besar).  
- Tidak Ada Autentikasi/Autorisasi Kompleks  
- Tanpa login, atau login sangat basic.  
- Tanpa Database Besar atau ORM  
- Atau hanya menggunakan file JSON, SQLite, atau array lokal.  
- Struktur Kode Masih Flat / Sederhana  
- Belum menggunakan arsitektur MVC kompleks atau framework besar seperti Laravel, Django fullstack, dll.  
- Tidak Responsive pada Semua Level  
- Tidak harus mobile-first atau ada dark mode, animasi kompleks, dsb.  
  
#### Contoh Web Programming Sederhana:  
- Kalkulator online (HTML/JS)  
- Form validasi (JS native)  
- REST API dengan Express.js (tanpa DB atau hanya JSON)  
- Website anime list statis  
- Landing page dengan Bootstrap  
- PHP-native CRUD (tanpa framework)  

#### Kalau proyek kamu mulai punya:  
- Autentikasi pengguna  
- Upload file  
- Middleware, routing kompleks  
- SPA (Single Page App) pakai React/Vue + state management  
- Integrasi database besar (MySQL, MongoDB)  
- Penggunaan API eksternal  
Maka sudah mulai masuk menengah.
